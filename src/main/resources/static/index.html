Avvia setup del database

            
Prepara il container oracle-corso per il workshop, via
               docker exec + sqlplus. Vedrai ogni comando in un terminale.

            
Cosa succederà:

            
    
                
  • Verifica connessione e versione del database
    • 
                
  • Creazione utente tuner con i privilegi del corso (abstract §6.5)
    • 
                
  • Grant su dizionario/V$ e sugli schemi HR e SH (a corso e tuner)
    • 
                
  • Raccolta statistiche ottimizzatore su HR e SH
    • 
                
  • Caricamento dati di test in bench_customers + statistiche
    • 
                
  • Verifica finale (utente e schemi)
    • 
            

            
⚠️ L'operazione ricrea l'utente tuner e
               ripopola bench_customers (TRUNCATE). Gli schemi
               HR e SH devono già essere installati (si installano a mano nel container):
               se mancano, il setup si blocca con le istruzioni. Se annulli, non accade nulla.

            

                
                
            

        

    


    
    

        

            

                
                
            

            
Setup del database

            

            

        

    


    
    

        

            
            
⚠️ Container Docker non attivo

            


            
Prima di avviare il setup, assicurati di:

            
    
                
  1. Avere Docker Desktop installato e in esecuzione.
    2. 
                
  2. Avere un container Oracle avviato con una delle due strade qui sotto.
    3. 
                
  3. Che application.yml punti a service name e credenziali corretti.
    4. 
            


            
Strada A — Community gvenzl (consigliata)

            
Oracle 26ai Free, avvio in pochi secondi, service name FREEPDB1.
               Con il docker-compose.yml del progetto (volume persistente):

            
docker compose up -d
docker logs -f oracle-corso    # attendi "DATABASE IS READY TO USE!"

            
Oppure senza compose:

            
docker run -d --name oracle-corso -p 1521:1521 -p 5500:5500 \
  -e ORACLE_PASSWORD=Corso2026 -e APP_USER=corso -e APP_USER_PASSWORD=Corso2026 \
  gvenzl/oracle-free:slim-faststart

            
application.yml (Strada A):

            
spring:
  datasource:
    url: jdbc:oracle:thin:@//localhost:1521/FREEPDB1
    username: corso
    password: Corso2026
oracle-bench:
  setup:
    container: oracle-corso
    service: FREEPDB1
    system-password: Corso2026
    app-password: Corso2026


            
Strada B — Oracle 19c ufficiale

            
Fedeltà 19c assoluta; richiede login su
               container-registry.oracle.com. Service name ORCLPDB1.

            
docker login container-registry.oracle.com
docker pull container-registry.oracle.com/database/enterprise:19.3.0.0
docker run -d --name oracle19c -p 1521:1521 -p 5500:5500 \
  -e ORACLE_SID=ORCLCDB -e ORACLE_PDB=ORCLPDB1 -e ORACLE_PWD=Corso2026 \
  -v oracle19c-data:/opt/oracle/oradata \
  container-registry.oracle.com/database/enterprise:19.3.0.0

            
application.yml (Strada B — nota container e service diversi):

            
spring:
  datasource:
    url: jdbc:oracle:thin:@//localhost:1521/ORCLPDB1
    username: corso
    password: Corso2026
oracle-bench:
  setup:
    container: oracle19c
    service: ORCLPDB1
    system-password: Corso2026
    app-password: Corso2026

            
Sull'immagine 19c ufficiale l'utente corso non esiste: crealo una volta
               come system — ALTER SESSION SET CONTAINER=ORCLPDB1;
               CREATE USER corso IDENTIFIED BY Corso2026 DEFAULT TABLESPACE users QUOTA UNLIMITED ON users;
               GRANT CREATE SESSION, CREATE TABLE TO corso; — poi il setup fa il resto.


            
✅ Il setup automatico funziona con entrambe le edizioni: cambia
               solo la configurazione qui sopra. Gli schemi HR/SH devono già esistere
               (il setup li configura, non li installa; SH richiede SQLcl).
